# 手机五线谱识谱识音 HTML App 开发计划

## Summary
- 目标：开发一个手机优先、可部署到 GitHub Pages 的静态 HTML 练习 app，用来训练高音谱号/低音谱号的基础识谱、音名、固定唱名和基础听音绑定。
- 交付形态：纯静态网页，无后端；文件为 `index.html`、`styles.css`、`app.js`，并在项目根目录保存本计划为 `开发计划.MD`。
- 核心体验：打开即练，不做落地页；主屏包含模式切换、谱面显示、播放按钮、答案按钮、即时反馈和练习统计。
- 设计方向：安静、清晰、适合碎片时间练习；大触控按钮、少干扰、手机竖屏优先，兼容桌面浏览。

## Key Changes
- 实现谱面渲染：用 Canvas 程序化绘制五线谱、谱号、音符、上下加线。
- 建立统一音符模型：以 `clef`、`staffStep`、`letter`、`solfege`、`octave`、`midi` 表示音符；固定唱名为 C=do、D=re、E=mi、F=fa、G=sol、A=la、B=si。
- 练习范围：v1 只覆盖自然音，不包含升降号、调号、节奏和拍号；高音谱、低音谱都支持五条线、四个间，以及上下各 4 条加线。
- 音频：使用 Web Audio 播放对应音高，A4=440Hz；声音由用户点击触发，避免浏览器自动播放限制。
- 进度：用 `localStorage` 保存正确率、连续答对、错题位置、最近练习模式；提供重置入口。

## Modes
- 随机抽查模式：随机显示高音谱或低音谱上的一个音，用户点选音名和唱名；答错时显示正确答案，并提高该位置后续出现概率。
- 规律填空模式：针对固定集合训练，例如高音谱五线 E/G/B/D/F、四间 F/A/C/E、低音谱五线 G/B/D/F/A、四间 A/C/E/G；用户逐格点选答案并即时校验。
- 加线专项模式：分别练高音谱/低音谱的上加线、下加线，覆盖 1 到 4 条加线，并可切换“只考线”或“线和间都考”。
- 听音绑定模式：显示谱面并播放音高，用户点选音名/唱名；重点是把视觉位置、音名、唱名和听觉连起来，不做纯听辨考试。
- 复习模式：从错题和低正确率位置生成短练习组，每组 10 题，结束后给出薄弱位置摘要。

## Interfaces And Behavior
- 主导航使用 segmented tabs：`随机`、`规律`、`加线`、`听音`、`复习`。
- 答案区使用大按钮：音名 `C D E F G A B`，唱名 `do re mi fa sol la si`；答完音名和唱名后自动校验。
- 设置区包含：谱号范围、高音/低音/混合、难度范围、是否自动播放、是否显示中文提示、是否复习错题。
- 每题反馈：正确显示绿色状态和下一题按钮；错误显示红色状态、正确音名/唱名、可重新听音。
- GitHub Pages 兼容：全部资源使用相对路径，不依赖构建步骤，不要求服务器 API。

## Test Plan
- 映射测试：验证高音谱五线为 E/G/B/D/F，四间为 F/A/C/E；低音谱五线为 G/B/D/F/A，四间为 A/C/E/G。
- 加线测试：验证高音谱、低音谱上下各 4 条加线的音名、唱名、八度和 MIDI 值正确。
- 交互测试：随机出题、点选答案、答错提示、下一题、切换模式、重置进度都能正常工作。
- 音频测试：点击播放后能听到对应音高；静音或浏览器限制时页面不崩溃，并给出可理解状态。
- 移动端测试：在 360x640、390x844、430x932 视口检查按钮不重叠、谱面不裁切、文字不溢出。
- 部署测试：直接打开本地 HTML 和 GitHub Pages URL 都能运行；刷新后统计保留。

## Assumptions
- 使用固定唱名，不做首调唱名。
- v1 不做用户账号、云同步、复杂课程系统或 MIDI 输入。
- 优先做点选答案，暂不做键盘输入；后续可加“默写模式”。
